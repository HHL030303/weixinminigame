# 速算 24 点（微信小游戏 / Canvas）

这是一个基于微信小游戏 Canvas 模式的“数学小游乐园”合集小游戏。

- **首页**：模块入口（卡通风格卡片），点击进入不同益智游戏。
- **当前已实现模块**：`速算 24 点`
  - 点击 4 个数字（每个数字只能用一次）→ 在下方点击运算符 `+ - * / ( )` → 四个数字用完后出现 `=` → 点击 `=` 统一结算，展示完整表达式与结果，并给出胜负反馈。

> 当前实现：段位/关卡/计时在运行时内存中计算；没有接入云端排行榜或本地持久化存储。

---

## 1. 项目目录结构

- `project.config.json`
  - 微信开发者工具工程配置（关键字段：`compileType: "game"`，以及 `miniprogramRoot` 指向 `minigame/`）。
- `minigame/`
  - `game.json`：小游戏窗口/状态栏/网络超时等全局配置。
  - `game.js`：入口与场景路由（首页、模块切换、公共绘制/输入）。
  - `modules/`
    - `game24.js`：速算24模块（状态、规则校验、渲染、交互）。
    - `findNumber.js`：找数字模块（随机块、顺序点击、倒计时、排行榜）。

> 现在两个游戏已拆为独立模块，后续新增游戏时只需新增一个 `modules/xxx.js` 并在 `game.js` 注册。

## 3. 玩法规则（交互）

### 首页
1. 进入小游戏后先到首页，展示多个游戏模块入口（如：速算24点、找数字等）。
2. 点击某个模块进入对应游戏场景；未实现的模块会弹出“敬请期待”提示。

### 速算 24 点
1. 进入速算24点后随机生成一关“4 个数字”（保证可解）。
2. 上方 4 个数字牌：
   - 每个数字可点击一次。
   - 点击后会追加到中间“算式条”里，同时该数字变为“已用”状态。
   - **不会出现“93”这种多位数**：两个数字之间**必须**插入运算符（或括号规则允许的位置），否则本次点击无效并弹出红色提示。
3. 下方运算符区域：
   - 默认按钮：`+ - * / ( )`
   - **不能连续点两个运算符**（如 `+-`、`++`），无效操作会提示且不写入算式。
   - 括号需配对；左括号只能出现在开头、运算符后或另一个左括号后；右括号只能跟在数字或右括号后。
   - 当且仅当四个数字都已用完（且仍在计时内），按钮区域会自动多出一个 `=`。
4. 点击 `=`：
   - 使用当前算式条内容求值（通过 `eval` 计算；运算顺序遵循 JavaScript 规则，即先乘除后加减，括号优先）。
   - 如果结果为 `24`（允许很小浮点误差），判定通关并展示“完整表达式 + 结果”。
   - 否则判定失败并展示表达式与结果。
5. **结算弹窗**：可点击右上角关闭、`知道了`、或**点击弹窗外的半透明区域**关闭；关闭后会自动进入下一题（重新开始一局）。

### 找数字 1-30
1. 进入后会出现 **1～30** 的不规则色块（随机位置，Canvas 绘制），并开始 **60 秒倒计时**。
2. 目标：在倒计时内按顺序点击 **1 → 2 → 3 → … → 30**。
3. 点击规则：
   - 点对（当前需要的数字）：色块高亮，并进入下一个目标。
   - 点错（不是当前目标数字）：会出现 **界面抖动 + 红色提示条**，该次点击不生效。
   - 已选中的数字再次点击：会 **取消选中**，并自动回到“最小未完成数字”作为下一目标。
4. **清除全部**：
   - 右下角有“清除全部”按钮，一键取消所有已选数字，目标回到 1。
5. 结束与得分：
   - 时间到或找齐 30 个后结束。
   - 得分 = 已选中数字数量（0～30）；用时用于同分排序（越快越靠前）。
6. **结果弹窗**：
   - 结果弹窗可点右上角关闭、点“再来一局”、或点弹窗外遮罩关闭；关闭后会立刻开始新一局。
7. **本地排行榜**：
   - 会把成绩写入本地存储（`wx.setStorageSync('findNumberLeaderboard')`），并在弹窗里显示前 3 名。

## 4. 计时、胜负与段位

### 倒计时
- 每关默认 `60` 秒倒计时（`ROUND_TIME = 60`）。
- 倒计时结束后进入失败状态（`timeout`）。

### 段位（基于通关次数）
- 通关次数使用 `state.score` 记录（当前实现为运行时内存，不持久化）。
- 称号梯度（按 `score`）：
  - `<= 1`：`初出江湖`
  - `<= 3`：`新鲜菜鸟`
  - `<= 6`：`速算达人`
  - `> 6`：`江湖传说`
- 当前关卡展示为：`score + 1`

### 关卡推进
- 点击底部 `换一题` 会直接重置为下一关的随机题目，并重置计时。
- 只有在点击 `=` 后且算出 24 时才会 `score++`。

## 5. 题目生成（保证可解）

核心逻辑位于 `minigame/game.js`：

- `isSolvable(nums, target)`：使用 DFS 枚举任意运算顺序与括号结构，判断是否存在表达式可得到目标值 `24`。
- `randomQuestion()`：随机生成 4 个数字（默认范围 `1~9`），重复采样直到找到可解题；兜底返回 `[3, 3, 8, 8]`。

> 注意：为了避免浮点状态爆炸，DFS 里对值做了适度量化/容差判断。

## 6. 代码入口与关键函数

### 入口
- `minigame/game.js` 顶部创建 canvas：
  - `const canvas = wx.createCanvas()`
  - `const ctx = canvas.getContext('2d')`

### 主要流程
- `setupCanvas()`：初始化 `state.w/state.h/state.dpr`，并配置 canvas 物理尺寸与坐标变换。
- `loop()`：每帧调用 `render()`。
- `render()`：根据 `state.scene` 渲染不同场景：
  - `home`：`renderHome()`
  - `game24`：`renderGame24()`

### 命中与交互
- `bindInput()`：注册 `wx.onTouchStart`。
- `onTap(x, y)`：
  - 在 `home`：命中首页卡片入口并跳转场景
  - 在 `game24`：命中返回按钮 / 换一题 / 数字牌 / 运算符区
- `onTapEqual()`：校验是否数字都用完 → 计算表达式 → 更新胜负面板与 `score`。

### 触摸坐标注意点（重要）
- 在 `bindInput()` 中对触摸坐标做了 `x / state.dpr`、`y / state.dpr` 的换算，确保命中检测与绘制坐标一致。

## 7. 当前限制与后续扩展建议

1. 本地排行榜/持久化：
   - 当前仅在运行时内存中记录 `score`，没有写入 `wx.setStorageSync` 或调用后端。
2. “按评分排名”：
   - 当前只展示段位称号，并未实现“排行榜列表”（例如保存多个历史成绩并排序）。
3. 更安全的表达式解析：
   - 当前用 `eval` 计算由按钮构造的表达式。未来可改为手写表达式解析/计算器以彻底避免任何潜在风险。

如果你希望我下一步把“排行榜（本地或云端）”补齐，我可以基于现有 `score` 和倒计时，设计一个更具体的“评分公式”和持久化结构。
